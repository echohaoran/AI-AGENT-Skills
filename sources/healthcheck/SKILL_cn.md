# 主机安全加固

> **来源：** 飞书机器人技能库  
> **API 依赖：** 是 | **协议：** MIT

## 何时使用

✅ **使用此技能的场景：**

- 需要安全审计
- 需要配置防火墙/SSH/更新加固
- 需要评估风险暴露
- 需要设置周期性安全检查
- 需要检查 OpenClaw 版本状态
- 用户提到"安全"、"加固"、"审计"

❌ **不要使用的场景：**

- 只需要简单系统信息（使用 system-info）
- 用户明确拒绝权限检查

## 核心规则

- **需要显式批准**：任何变更操作前必须确认
- **不修改远程访问**：除非确认用户连接方式
- **优先可逆变更**：分阶段修改，准备回滚方案
- **模型建议**：推荐使用先进模型（如 Opus 4.5），低于此级别会建议切换

## 工作流程

### 1. 建立上下文（只读）

获取以下信息：
1. 操作系统和版本
2. 权限级别
3. 访问路径
4. 网络暴露情况
5. OpenClaw 网关状态
6. 备份系统状态
7. 磁盘加密状态
8. 自动安全更新状态

### 2. 运行安全审计（只读）

```bash
openclaw security audit
openclaw security audit --deep
```

### 3. 检查版本状态

```bash
openclaw update status
```

### 4. 确定风险容限

| 风险配置 | 说明 |
|----------|------|
| 家庭/工作站平衡 | 防火墙开启，远程访问限制在局域网 |
| VPS 加固 | 默认拒绝防火墙，最小开放端口 |
| 开发者便利 | 允许更多本地服务，明确暴露警告 |
| 自定义 | 用户定义的约束 |

### 5. 生成修复计划

包含：
- 目标配置
- 当前状态
- 差距分析
- 逐步修复命令
- 访问保护策略
- 风险和潜在锁定场景

## 必须确认的操作

- 防火墙规则变更
- 端口开放/关闭
- SSH/RDP 配置变更
- 包安装/删除
- 服务启用/禁用
- 访问敏感文件或凭据

## 定期检查

建议定期运行：
```bash
openclaw security audit
openclaw update status
```

可使用 OpenClaw cron 安排周期性审计：
```bash
openclaw cron add --name healthcheck:security-audit
```
