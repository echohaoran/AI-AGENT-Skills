SKILL.md
Skill Vetter 🔒

AI 代理技能的安全优先审查协议。在安装任何技能之前务必先审查。

何时使用

    从 ClawdHub 安装任何技能之前
    从 GitHub 仓库运行技能之前
    评估其他代理共享的技能时
    被要求安装未知代码的任何时候

审查协议
第 1 步：来源检查

需要回答的问题：
- [ ] 这个技能来自哪里？
- [ ] 作者是否知名/可靠？
- [ ] 有多少下载量/星标？
- [ ] 最后更新是什么时候？
- [ ] 有其他代理的评价吗？

第 2 步：代码审查（强制）

阅读技能中的所有文件。检查以下危险信号：

🚨 如果发现以下情况立即拒绝：
─────────────────────────────────────────
• 向未知 URL 发送 curl/wget 请求
• 向外部服务器发送数据
• 请求凭据/令牌/API 密钥
• 无明确原因读取 ~/.ssh、~/.aws、~/.config
• 访问 MEMORY.md、USER.md、SOUL.md、IDENTITY.md
• 对任何内容使用 base64 解码
• 使用 eval() 或 exec() 处理外部输入
• 修改工作区外的系统文件
• 安装未列出的软件包
• 向 IP 而非域名发起网络调用
• 混淆代码（压缩、编码、压缩过）
• 请求提升权限/sudo 权限
• 访问浏览器 cookies/会话
• 接触凭据文件
─────────────────────────────────────────

第 3 步：权限范围评估

评估内容：
- [ ] 它需要读取哪些文件？
- [ ] 它需要写入哪些文件？
- [ ] 它运行哪些命令？
- [ ] 它需要网络访问吗？访问哪里？
- [ ] 范围是否符合其声明的目的？

第 4 步：风险分类
风险等级	示例	操作
🟢 低	笔记、天气、格式设置	基本审查，可以安装
🟡 中等	文件操作、浏览器、API	需要完整代码审查
🔴 高	凭据、交易、系统	需要人工批准
⛔ 极高	安全配置、根访问	不要安装

快速审查命令

对于 GitHub 托管的技能：

# 检查仓库统计
curl -s "https://api.github.com/repos/OWNER/REPO" | jq '{stars: .stargazers_count, forks: .forks_count, updated: .updated_at}'

# 列出技能文件
curl -s "https://api.github.com/repos/OWNER/REPO/contents/skills/SKILL_NAME" | jq '.[].name'

# 获取并审查 SKILL.md
curl -s "https://raw.githubusercontent.com/OWNER/REPO/main/skills/SKILL_NAME/SKILL.md"

信任层级

    官方 OpenClaw 技能 → 较低审查（仍需审查）
    高星标仓库（1000+） → 中等审查
    知名作者 → 中等审查
    新/未知来源 → 最高审查
    请求凭据的技能 → 始终需要人工批准

记住

    没有任何技能值得牺牲安全
    如有疑问，不要安装
    高风险决策请询问人工
    记录审查内容以便将来参考
