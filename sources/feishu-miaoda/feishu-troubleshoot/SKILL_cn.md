# 飞书插件问题排查

> **来源：** 飞书机器人技能库  
> **API 依赖：** 否 | **协议：** MIT

## 何时使用

✅ **使用此技能的场景：**

- 卡片按钮点击无反应
- 多次授权仍失败
- 自动授权无法解决问题
- 需要检查账户配置、API 连通性、应用权限
- 用户提到"出错了"、"不能用"、"报错"

❌ **不要使用的场景：**

- 常规功能咨询（使用对应功能的技能）

## 常见问题

### 卡片按钮点击无反应

**现象**：点击卡片按钮后没有反应，然后提示报错

**原因**：应用未开通「消息卡片回传交互」权限

**解决步骤**：
1. 登录飞书开放平台：https://open.feishu.cn/app
2. 选择应用 → **事件与回调**
3. 修改订阅方式为"长链接"
4. 添加回调 "卡片回传交互"（card.action.trigger）
5. 创建应用版本 → 提交审核 → 发布

## 诊断命令

诊断命令 `/feishu doctor` 用于排查复杂权限问题

### 使用方法
在飞书聊天中直接输入（作为用户消息）：
```
/feishu doctor
```

### 检查项目

- **📋 诊断摘要**：总体状态、问题列表
- **环境信息**：插件版本
- **账号信息**：凭证完整性、Bot 信息
- **应用权限**：已开通权限、缺失权限
- **用户授权**：Token 状态、权限对照
