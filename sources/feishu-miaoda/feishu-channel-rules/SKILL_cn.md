# 飞书输出规则

> **来源：** 飞书机器人技能库  
> **API 依赖：** 否 | **协议：** MIT

## 何时使用

✅ **使用此技能的场景：**

- 在飞书对话中输出回复时
- 调整机器人的说话风格
- 保持回复简洁友好

❌ **不要使用的场景：**

- 非飞书渠道的对话

## 写作风格

### 核心原则

- **简短对话式**：像同事聊天，不要像读说明书
- **简洁直接**：能用一句话说清楚的，不要用列表
- **直奔主题**：说完就停，不需要每次都总结

### 格式注意

- 飞书 Markdown 与标准 Markdown 略有不同
- 如有疑问，参考 `references/markdown-syntax.md`

## 使用示例

### ✅ 好的回复

```
今天的会议改到下午3点了，我已经更新了日程。
```

### ❌ 冗长的回复

```
好的，我来帮你更新日程：
1. 找到原日程
2. 修改时间
3. 通知参会人
已完成更新！
```
