# 妙搭应用开发

> **来源：** 飞书机器人技能库  
> **API 依赖：** 是 | **协议：** MIT

## 何时使用

✅ **使用此技能的场景：**

- 信息可视化展示（图表、看板、简报、报表、数据大屏）
- PPT/演示文稿
- 简单工具原型
- 用户提到"妙搭"、"vibe coding"
- 用户明确说做"原型/demo/展示页"

❌ **不要使用的场景：**

- **CRM/ERP/OA 等管理系统**（用户主动说做原型除外）
- **AI 对话/智能体/助手**（核心功能是 AI 对话交互）
- **定时任务/自动化/推送**
- **知识库/智能问答**
- **纯文字创作**（写文章、脚本、文案）
- **纯数据分析**（只要分析结论，不做可视化）
- **复杂 PDF 生成**
- **需要外部数据源**（实时行情、API 对接）

## 支持的四类场景

1. **信息可视化展示**：图表、看板、简报、报表、数据大屏
2. **简单应用原型**：落地页、概念演示、方案展示
3. **PPT/演示文稿**：幻灯片、课件、汇报材料
4. **简单工具**：有页面和基础增删改查，不涉及复杂业务流程

## 不适用场景

| 类别 | 关键词 |
|------|--------|
| AI 对话/智能体 | "AI助手"、"智能客服"、"智能体" |
| 定时任务 | "每日"、"定时"、"自动采集" |
| 知识库 | "知识库"、"文档问答"、"智能搜索" |
| 复杂系统 | "CRM"、"ERP"、"OA"、"管理系统" |

## 边界模糊时判断

1. 需求命中不适用场景表？
2. 是复杂管理系统吗？用户主动说做原型除外
3. 是简单工具还是真正的业务系统？

## 执行流程

### 1. 能力边界判断

先判断是否在支持范围内：
- 是 → 继续
- 否 → 输出引导文案，引导去妙搭主站

### 2. 判断是否需要前置工作

- 纯建站请求 → 直接派活
- 有上下文需要传递 → 先调 `miaoda_write_reference`

### 3. 派活给妙搭

通过 `sessions_spawn` 派给 subagent，只传：
- runtime: "subagent"
- mode: "run"
- task: 生成指令

### 4. 返回结果

回复用户"交给妙搭了，稍等"，然后等待 subagent 返回。

## 引导文案格式

```
这个需求比较复杂，你可以去妙搭搭建试试：

[去妙搭创建<需求描述>](https://miaoda.feishu.cn/home?userInput=<URL编码的需求>&mode=application&open-from=openclaw)
```
